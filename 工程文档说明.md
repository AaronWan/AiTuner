## 一、项目概述

基于 Android 的智能吉他调音器，核心功能：
1. **吉他调音器** — 麦克风录音 → FFT 自相关音高检测 → 半圆仪表盘显示
2. **AI 对话助手** — 可插拔 LLM Provider，支持流式 SSE
3. **唱谱练习** — AI 生成乐谱 → 用户跟唱 → 逐音符评分
4. **简谱显示** — 支持简谱/音名一键切换
5. **多语言** — 简体中文(默认)、English、한국어、繁體中文


### 3.3 已知编译坑

1. **JDK 21 → jlink 报错**：AGP 8.2.0 的 `core-for-system-modules.jar` 与 JDK 21 jlink 不兼容，必须用 JDK 17
2. **kapt 在 JDK 21 报 IllegalAccessError**：换 KSP（无模块系统限制）
3. **Compose BOM 2024.01.00 + CircularProgressIndicator** → `NoSuchMethodError`：升级 BOM 到 2024.02.00 修复
4. **`R` 类引用**：非 `com.example.aituner` 包下的文件需要 `import com.example.aituner.R`

## 六、核心架构决策

### 6.1 音高检测算法

**自相关法** + **抛物线插值**（`PitchDetector.kt`）

- 采样率 44100Hz，缓冲区 4096 样本（≈93ms）
- 检测范围 60Hz(E2) ~ 500Hz(B4+)
- 信号能量阈值 0.0001 → 静音判断
- 置信度 = 能量得分×0.4 + 相关系数得分×0.6

### 6.2 AI Provider 可插拔设计

```
AiProvider (interface)
  ├── chat()       → Result<String>  (阻塞式)
  └── chatStream() → Flow<String>    (SSE流式)
       ├── OpenAIProvider:  POST /v1/chat/completions  stream=true
       │                     解析 SSE data: {...}
       ├── ClaudeProvider:  流式 fallback 到 chat()
       └── OllamaProvider:  流式 fallback 到 chat()
```

### 6.3 乐谱生成流程

```
用户输入 → SingPracticeViewModel.generateScore()
  → AiProvider.chatStream(system_prompt + user_request)
    → SSE 累积 tokens
      → ScoreParser.parse() 解析为 ScoreNote 列表
        → 全屏沉浸式展示（2列网格 / 大字体）
```

AI 提示词约束：16-32 音符，A3-C5 范围，1-2s 时长，格式 `NoteName:Duration`。

### 6.4 唱谱评分标准

| 偏移范围 | 判定 |
|----------|------|
| ±10 音分 | 完美 (PERFECT) |
| ±25 音分 | 还行 (GOOD) |
| >25 偏高 | 偏高 (SHARP) |
| >25 偏低 | 偏低 (FLAT) |
| 无声 | 错过 (MISSED) |
